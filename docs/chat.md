# Chat & Conversation Features

This document describes the chat-related endpoints in the Osijek AI Guide API.

## Core Chat Endpoints

### `POST /chat`
Full chat with Lega, with complete tool-calling memory across turns.

**Authentication:** Required (JWT)

**Body:**
```json
{
  "message": "Koje dogaÄ‘aje ima veÄŤeras?",
  "language": "hr",
  "max_history": 20
}
```

### `POST /chat/stream`
Same as `/chat` but returns Server-Sent Events (recommended for mobile apps).

---

## Conversation Management

### `POST /chat/reset`
Reset the authenticated user's chat history. Preferences are preserved by default.

**Authentication:** Required

**Query params:**
- `keep_preferences` (default: true)

### `POST /chat/history/{user_id}/reset`
Admin-style endpoint to reset any user's chat history.

**Authentication:** Not required (but should be protected in production)

**Query params:**
- `keep_preferences`

### `DELETE /chat/history/{user_id}`
Deletes the entire chat history for a user (legacy).

---

## Summary & Insights

### `GET /chat/history/{user_id}/summary`
Returns an **LLM-generated summary** of the conversation (much better than simple message counting).

**Query params:**
- `format`: `short` | `medium` | `detailed` (default: medium)
- `max_messages`: How many recent messages to analyze (default: 30)

**Response includes:**
- `summary` (generated by LLM)
- Basic stats (message counts, last messages)
- `analyzed_messages`

---

## Metrics

### `GET /chat/metrics`
Returns tool usage statistics for the authenticated user.

**Query params:**
- `include_global`: If true, also returns global top tools

**Response:**
```json
{
  "your_tool_usage": {
    "search_osijek_events": { "count": 47, "last_used": "..." },
    ...
  },
  "total_tool_calls": 69
}
```

### `GET /admin/metrics/tool_usage`
Global tool usage statistics (protected).

---

## Feedback

### `POST /chat/feedback`
Submit feedback on a specific message in your chat history.

**Body:**
```json
{
  "message_index": 7,
  "rating": 1,           // 1 = thumbs up, -1 = thumbs down
  "comment": "OdliÄŤna preporuka!"
}
```

### `GET /chat/feedback`
Returns all feedback given by the authenticated user + summary.

---

## Personalization

User preferences are managed via:

- `GET /user/me`
- `POST /user/me/preferences`

These preferences are automatically injected into the system prompt for better personalization.

---

## Best Practices for Mobile Apps

- Use `/chat/stream` for smooth UX.
- Use `/chat/reset` when user wants to start fresh.
- Call `/chat/summary` periodically to show conversation overview.
- Allow users to give feedback via `/chat/feedback`.
- Use `/chat/metrics` to show users how they interact with Lega.

---

## Notes

- All chat history is stored per `user_id`.
- When using JWT authentication, the user ID from the token is preferred.
- The system supports full tool-calling memory across multiple turns.
